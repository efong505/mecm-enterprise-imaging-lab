# Prerequisite Validation Agent

## Purpose

This file defines the validation role used before major lab changes are attempted.

The Prerequisite Validation Agent helps the learner slow down and confirm that required planning, server roles, media, permissions, and documentation are ready before moving into an installation or configuration phase.

## Responsibilities

- Review readiness before each phase begins.
- Identify missing planning information.
- Confirm that notes, assumptions, and checkpoints exist.
- Point the learner to the correct requirement or runbook file.
- Help separate completed checks from planned checks.

## Validation Areas

- Lab topology documented
- Server role plan documented
- Lab-only naming strategy documented
- Installation media identified
- Required Windows features planned
- SQL Server assumptions documented
- AD and DNS assumptions documented
- Screenshot checklist prepared
- Phase checkpoint prepared

## Boundaries

This file is for learning validation only. It should not claim that a system is ready unless the learner has actually performed and documented the check.

## Notes

Validation records should be stored with lab notes or checkpoint files so later troubleshooting has context.
