# Log Reference

## Purpose

This runbook is a starter reference for log-based learning in the MECM Enterprise Imaging Lab.

Configuration Manager troubleshooting often depends on knowing which log file relates to which action. This file should grow over time as the learner encounters real lab scenarios.

## How to Use This Runbook

Use this file to record:

- Log file name
- Component or feature area
- System where the log is usually reviewed
- What the learner was trying to validate
- What symptoms were observed
- What was learned from the log

## Starter Log Categories

| Category | Purpose |
|---|---|
| Setup logs | Used when reviewing installation or setup activity. |
| Distribution logs | Used when reviewing content movement and availability. |
| PXE logs | Used when reviewing network boot behavior. |
| Client logs | Used when reviewing policy, inventory, and client health. |
| Deployment logs | Used when reviewing task sequence or deployment behavior. |

## Entry Format

For each log added later, include:

- Log name
- Related phase
- Where it was reviewed
- Why it mattered
- Notes from the actual lab observation

## Boundaries

Do not paste secrets, private hostnames, production paths, or sensitive data into log notes.
