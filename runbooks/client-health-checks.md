# Client Health Checks

## Purpose

This runbook provides a starter structure for reviewing MECM client health in the practice lab.

The goal is to help the learner document client status clearly after a test client is added to the lab.

## Health Areas to Review

- Client installation status
- Policy retrieval behavior
- Collection membership
- Inventory reporting
- Client activity status
- Relevant client logs
- Recent deployment activity

## Suggested Record Format

For each client health check, record:

- Date of review
- Test client name
- Related phase
- What was expected
- What was observed
- Logs reviewed
- Follow-up actions

## Questions to Ask

- Is the client visible in the console?
- Is the client assigned to the expected site?
- Has the client received policy?
- Is inventory reporting expected or pending?
- Are there errors that need a troubleshooting entry?

## Boundaries

Do not include real user data or production endpoint information. Use lab-only test systems.
